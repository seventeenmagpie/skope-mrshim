HardwareReport(...)
Returns status attributes of each channel of the amplifers.
Including attributes such as status codes, and internal temperatures.
If no communication could be made with the amplifier then the only
key in the returned dictionary is "connected".
 
Returns

Result : Dict
    Amplifier channel attributes:
        - 'connected': 1 when amplifier is detected

        - 'status': List of status codes for diagnosis
        - 'curr': List of measured current values
        - 'temp': List of internal temperatures (on-board amplifiers)

LinkupHardware(...)
Connect to hardware devices. Continually retries to connect, if
a connection could not be established.
If a hardware input could not be detected, then no hardware input
is used.
 
Parameters
----------
ifname : string
    Name of the network interface device. See `JupiterDeviceInfo`.
mode : int
    0 For fast-mode (does not ramp to the shim values but
    applies the currents immediately).
    1 For safe-mode (ramps to the applied shim values).
opts : dict, optional
    Options for specifying the hardware input. If this is empty
    or None, then no input is used. The input can be used for
    trigger-based or real-time shimming.
    opts['pin'] - The pin number of the device. For Windows, this
                  values is between 1 and 3 (typically 3).
    opts['mode'] - 0 Rising-edge triggered
                 - 1 Falling-edge triggered
                 - 2 Analog mode
    If opts['mode'] is 0 or 1, then everytime a event is triggered
    the next set of shim values is applied to the amplifiers
    (see `LoadShimSets`).
    If opts['mode'] is 2, then the input is treated as an analog
    input and every time the input is sampled, the function
    `ShimRealtime` if called from the `shimplugin.dll\.so` shared
    library.
LoadShimSets(...)
Load a file with the table of shim currents to be applied (in Amps).
The values in the first row of the text file immediately be applied.
 
Parameters
----------
filename : string
    File path of the text file with the shim current values.
    The first row of values will be applid, then the next rows can
    be applied using a trigger (if the hardware input was set to either
    'rising edge' or 'falling edge' mode, see `LinkupHardware`), or by
    calling the function `ApplyShimManually`.
delay : int, optional
    Duration to wait (in ms) before applying the next set of shim values.
    Default: 0.
 
Raises
------
Exception when the text file cannot be parsed. Must be a space delimited
text file of float values.
 
NOTE
----
This will only be used when the hardware input is not in 'Analog' mode,
see `LinkupHardware`. If the hardware input was configured to 'Analog'
mode, then the 'shimplugin.dll/.so' shared library is used.
